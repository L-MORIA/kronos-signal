Kronos Signal — архитектура

Полная схема работы: от двойного клика до прогноза и интерпретации

🔵 Поток вызова (5-min версия)

  • Двойной клик → Kronos Signal.lnk → launch.bat → desktop_launcher.py → subprocess(run.py)
  • run.py читает config.yaml (SBERP, 1min→5min, sample_count=1, T=1.0, pred_len=24)
  • fetch_moex(): 2 запроса к MOEX ISS (вчера + сегодня), from+till, 1-мин свечи, ~1 сек
  • resample(): pandas → 5-мин OHLCV. fill_gaps: last-price-ffill, ~0.1 сек
  • KronosPredictor.predict(): 24 авторегрессивных шага, ~4 сек/шаг = ~98 сек
  • signal() + interpret(): сравнение forecast vs last close, порог 2%, текстовый разбор
  • Результат: stdout + log.txt

🟠 Поток вызова (Hourly версия)

  • Двойной клик → Kronos Signal Hourly.lnk → launch2.bat → desktop_launcher2.py → subprocess(run.py --config config-hourly.yaml)
  • run.py читает config-hourly.yaml (SBERP, 1h→1h, sample_count=3, T=0.85, top_p=0.85)
  • fetch_moex(): 1 запрос к MOEX ISS, 1-часовые свечи, 500 шт. = 14+ дней истории
  • predict(): 24 шага × 3 Monte Carlo = ~4 мин
  • MC усреднение сглаживает шум, T=0.85 делает прогноз чуть жёстче
  • Результат: stdout + log-hourly.txt

📦 Где что лежит

  • C:\kronos-models\ — Kronos-mini (16 MB) + Tokenizer-2k (15 MB). Только диск, не интернет
  • D:\kronos-signal\ — configs, scripts, venv, logs, kronos-source
  • update_models.bat — проверка обновлений на HuggingFace Hub. Запускать руками когда угодно
  • При каждом запуске run.py — интернет НЕ нужен. Модель локальная. fetch_moex() только при старте
